## 粘贴给 Agent 的完整提示词：

```
帮我创建一个迷宫游戏的前端页面，放在 frontend/maze-game/ 目录下。

## 要求：

### 文件结构
在 frontend/maze-game/ 下创建以下文件：
- index.html（页面结构）
- style.css（样式）
- maze.js（迷宫渲染 + 操控逻辑）

### ⚠️ 接口规范（必须严格按此格式调用，否则后端收不到！）

你负责调用后端接口，不需要实现后端。以下是你需要调用的接口：

**接口1：获取迷宫 [队员C 实现]**
请求：GET http://localhost:5001/api/maze/generate?width=21&height=21

响应格式（必须按此格式解析）：
{
  "maze": [[0,1,0,0,1,...], [1,0,0,1,0,...], ...],
  "width": 21,
  "height": 21
}

字段说明：
- maze: 二维数组，0=路，1=墙
- width: 迷宫宽度（列数）
- height: 迷宫高度（行数）
- 起点固定: maze[1][1]，终点固定: maze[height-2][width-2]
- width 和 height 必须是奇数

**接口2：提交成绩 [队员D 实现]**
请求：POST http://localhost:5002/api/score
Content-Type: application/json

请求体格式（必须严格一致）：
{
  "player": "玩家名",
  "time": 35,
  "level": 1
}

响应格式：
{
  "success": true,
  "message": "成绩已保存"
}

字段说明：
- player: 字符串，玩家名称
- time: 整数，通关用时（秒）
- level: 整数，关卡编号

### 功能需求：
1. 用 Canvas 画布渲染迷宫，画布大小 500x500，迷宫格子大小根据 width 自适应
2. 页面加载时自动调用 GET http://localhost:5001/api/maze/generate?width=21&height=21 获取迷宫
3. 玩家用键盘方向键 ↑↓←→ 控制移动，每次移动一个格子，只能在路(0)上走，不能穿墙(1)
4. 玩家用绿色圆形表示，起始点在 (1,1)，终点在 maze[height-2][width-2]，用红色标记
5. 到达终点后弹出提示"恭喜通关！"，然后调用 POST http://localhost:5002/api/score 提交成绩
   - player 字段用输入框让玩家填名字
   - time 为实际通关秒数
   - level 固定为 1
6. 页面上方显示计时器（从第一次按键开始计时，到通关停止）
7. 有"重新生成迷宫"按钮，点击后重新调用接口获取迷宫
8. 起点固定在 (1,1)，终点固定在右下角

### 样式要求：
- 页面居中，深色背景
- 迷宫墙壁用深灰色，道路用白色
- 玩家用绿色圆形，终点用红色方块
- 界面简洁美观

### 用到的数据结构（在注释里标注）：
- 二维数组：存储迷宫地图
- 在 maze.js 里用注释标注 "/* 数据结构：二维数组/矩阵，存储迷宫每个格子的值 */"

请一次性生成所有完整代码，不要省略。
```
